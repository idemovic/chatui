chatui integration guide npm install + static config

Embed chatui in your app

This guide walks you through installing chatui from an npm tarball (or local file path), mounting the widget in your React app, and configuring every option statically in code. No runtime config fetch, no settings UI needed.

1. Prerequisites

chatui ships as a pre-built ESM bundle with its CSS pre-compiled. The host needs:

  • React 19 (peer dependency)
  • Any modern bundler that supports ESM and JSON package exports (Vite, Webpack 5+, esbuild, Rollup, etc.)
  • Node 18 or newer (for installing the package)
No Tailwind required. dist/chatui.css is shipped pre-compiled, so your host doesn't need to know that chatui internally uses Tailwind v4.

2. Install

Option A: from a tarball (recommended for sharing)

Run npm pack in the chatui repo to produce chatui-X.Y.Z.tgz. Then in your host:

npm install /absolute/path/to/chatui-0.0.1.tgz

Option B: local file path (recommended for parallel development)

In your host's package.json:

{
  "dependencies": {
    "chatui": "file:../chatui"
  }
}

Then run npm install. Edits in the chatui repo reflect in the host after a dev-server restart.

Minimal host package.json

chatui's bundle already includes i18next, react-i18next, react-markdown, remark-gfm, zustand, and the pre-compiled CSS. Don't redeclare them in your host — doing so risks duplicate copies. The host needs only React, the bundler, and chatui itself: 

{
  "scripts": {
    "dev":   "vite",
    "build": "tsc -b && vite build"
  },
  "dependencies": {
    "chatui":    "file:../chatui/chatui-0.0.1.tgz",
    "react":     "^19.0.0",
    "react-dom": "^19.0.0"
  },
  "devDependencies": {
    "@types/react":        "^19.0.0",
    "@types/react-dom":    "^19.0.0",
    "@vitejs/plugin-react": "^6.0.0",
    "typescript":          "~5.9.0",
    "vite":                "^8.0.0"
  }
}

Add back only if you need them in your own host code (not for chatui to work):

PackageAdd when
i18next + react-i18nextYou want to register extra locales at runtime (see Languages section) or use useTranslation in your own components.
zustandYour host code has its own Zustand stores.
tailwindcss + @tailwindcss/viteYour host's own UI uses Tailwind. chatui doesn't need it — its CSS is already compiled.

3. Minimal mount

A single entry file is enough. The widget reads its configuration from chatui's Zustand store, so you set the config once before React renders and everything picks it up from there. 

// src/main.tsx
import { StrictMode } from 'react'
import { createRoot } from 'react-dom/client'

import { App as ChatWidget } from 'chatui'
import { useSettingsStore }    from 'chatui/store'
import                            'chatui/css'

// Configure BEFORE the first render. setConfig merges into the persisted store.
useSettingsStore.getState().setConfig({
  webhookUrl: 'https://your-n8n.example.com/webhook/abc123',
  mode:        'window',
  botName:     'Support',
  streaming:   true,
})
useSettingsStore.getState().setTheme('midnight')
useSettingsStore.getState().setLanguage('en')

const rootEl = document.getElementById('root')
if (!rootEl) throw new Error('Root element not found')

createRoot(rootEl).render(
  <StrictMode>
    {/* The chat panel needs a full-height parent. */}
    <div style={{ height: '100vh' }}>
      <ChatWidget />
    </div>
  </StrictMode>,
)
About persistence. chatui persists its config to localStorage under the key chatui-settings. Calling setConfig on boot overwrites the matching fields. Unset fields keep their stored values. If you want a fully clean slate, call localStorage.removeItem('chatui-settings') before calling setConfig.

4. Full static configuration

Every option available, with defaults and realistic values. Drop this into the bootstrap file and delete the fields you do not need. 

import { useSettingsStore } from 'chatui/store'

useSettingsStore.getState().setConfig({

  // ─── Required ────────────────────────────────────────────────────
  webhookUrl: 'https://your-n8n.example.com/webhook/abc123',

  // ─── Webhook transport ────────────────────────────────────────────
  webhookConfig: {
    method:  'POST',                           // or 'GET'
    headers: { 'X-API-Key': 'your-secret' },
  },
  chatInputKey:   'chatInput',                 // body key for the user message
  chatSessionKey: 'sessionId',                 // body key for the session id
  metadata:       { source: 'marketing-site' },  // merged into every request body

  // ─── Behavior ─────────────────────────────────────────────────────
  streaming:             false,                      // expect SSE from n8n
  showWelcomeScreen:     true,
  allowFileUploads:      false,
  allowedFilesMimeTypes: 'image/*,application/pdf',

  // ─── Layout ───────────────────────────────────────────────────────
  mode:        'window',                         // 'fullscreen' | 'window' | 'mixed'
  showSidebar: false,                                // fullscreen mode only

  // ─── Appearance ───────────────────────────────────────────────────
  theme:     'midnight',                         // see Themes section
  botName:   'Assistant',                        // global fallback (per-lang overrides below)
  botAvatar: 'amara',                            // built-in id, URL, or data URL. See Avatars section.

  // ─── CTA popup (window mode) ─────────────────────────────────────
  showCta:  true,
  ctaText:  'Hi! How can I help?',                // global fallback
  ctaDelay: 5000,                                    // ms before it appears
  ctaSound: true,                                    // Web Audio beep
  toggleButtonIcon: 'priya',                      // replaces the default chat bubble. Same value forms as botAvatar.

  // ─── UI lock ──────────────────────────────────────────────────────
  hideSettings: true,                                // default. Set to false to show the gear, theme picker, and settings modal.

  // ─── "Powered by" footer ─────────────────────────────────────────
  poweredByLabel: 'ELIA AI Assistant',            // link text shown after the localized "Powered by" prefix
  poweredByUrl:   'https://www.elia-asistent.com', // link target
  poweredByHide:  false,                             // hide the entire footer line

  // ─── i18n ─────────────────────────────────────────────────────────
  defaultLanguage:  'en',
  initialMessages: ['Hello! How can I help?'],    // global fallback
  i18n: {
    en: {
      botName:         'Sam',
      welcomeSubtitle: 'Ask me anything.',
      initialMessages: ['Hi! How can I help?', 'I speak English.'],
      ctaText:         'Try me now!',
    },
    sk: {
      botName:         'Sam',
      welcomeSubtitle: 'Opýtajte sa ma čokoľvek.',
      initialMessages: ['Dobrý deň! Ako vám môžem pomôcť?'],
      ctaText:         'Vyskúšajte ma!',
    },
  },
})

// Theme and active language are separate state slices
useSettingsStore.getState().setTheme('cosmos')
useSettingsStore.getState().setLanguage('sk')

7. Avatars and toggle button icon

chatui ships 12 built-in avatars inlined as data URLs inside dist/index.js. You can reference one by id, supply your own URL, or paste a data URL.

Both botAvatar and toggleButtonIcon accept the same value forms:

  • Built-in id: 'amara', 'david', 'elena', 'kenji', 'luca', 'marcus', 'mei', 'nadia', 'omar', 'priya', 'samuel', 'sofia'
  • URL: any http(s)://… or root-relative path such as '/img/my-avatar.png'
  • Data URL: 'data:image/png;base64,…' (what the built-in file picker produces)

If the settings UI is visible (hideSettings: false), end users can pick a built-in, upload a file (max 500 KB), or paste a URL from the settings modal. The toggle button icon picker only appears in window / mixed mode.

8. Languages

English and Slovak ship pre-bundled inside node_modules/chatui/dist/chunks/translation-*.js. The widget loads them on demand via dynamic import(), no HTTP fetch.

To register a new language at runtime (without rebuilding chatui):

import i18n from 'i18next'

i18n.addResourceBundle('fr', 'translation', {
  welcome:  { subtitle: 'Commencez une conversation.' },
  input:    { placeholder: 'Tapez un message...' },
  // ... full keys: copy from node_modules/chatui/dist/chunks/translation-*.js
})

useSettingsStore.getState().setLanguage('fr')

Then add any French per-language overrides under config.i18n.fr.

9. Gotchas and troubleshooting

"Calling require for 'react' in an environment that doesn't expose require"

Vite 8 (Rolldown) pre-bundles node_modules packages by default. For chatui that produces a broken CJS-style interop wrapper for the externalized React peer dep. Since dist/index.js is already a finished ESM bundle, tell Vite to skip pre-bundling it: 

export default defineConfig({
  plugins: [react()],
  optimizeDeps: {
    exclude: ['chatui'],
  },
})

Then clear the Vite cache and restart:

rm -rf node_modules/.vite   # or on Windows: rmdir /s /q node_modules\.vite
npm run dev

"Invalid hook call" or two React copies

chatui declares react and react-dom as peerDependencies. If npm installs two copies anyway (because of version mismatch), dedupe: 

npm dedupe
# or if that does not help, add to your package.json:
"overrides": { "react": "^19.0.0", "react-dom": "^19.0.0" }

Webhook is called but returns CORS errors

Your n8n Chat Trigger must allow the origin your host page is served from. Open the Chat Trigger node in n8n and add your origin under Allowed Origins (CORS).

Showing the settings gear (it is hidden by default)

For npm consumers the gear, theme picker, and settings modal are hidden out of the box, because your config is static code. If you want them back (admin dashboard, internal tools, live demo), flip the flag: 

useSettingsStore.getState().setConfig({
  webhookUrl:   'https://your-n8n/webhook/abc',
  hideSettings: false,
})

The widget renders but has no height

chatui uses h-full internally. Its parent container must have an explicit height, for example height: 100vh or a fixed pixel value.

Clearing persisted state during development

// DevTools console, or a temporary boot-time line
localStorage.removeItem('chatui-settings')
localStorage.removeItem('chatui-chat')
